# Keysplit
A command-line tool for splitting Ethereum validator keys for distributed validation on the SSV network.

# Overview
Keysplit is a specialized tool designed to facilitate the distribution of Ethereum validator responsibilities across multiple operators. By splitting a validator's secret key into encrypted shares, the tool enables distributed validation where multiple operators can collectively perform validation duties without any single party having access to the complete key.
The tool works with keystore files generated by the deposit CLI.

# Features
- Two methods of key splitting
    - **Manual**: Manually input all required fields for splitting
    - **Onchain**: Scrape blockchain data to automatically retrieve information needed for splitting
- Support for committees of 4, 7, 10, or 13 operators (enforced by design)
- RSA encryption of keyshares with operator public keys
- Standardized output format compatible with SSV network requirements

# How It Works
Keysplit follows a specific process to securely split and distribute validator keys:

1) Read and parse the keystore file containing the validator's encrypted key
2) Extract the validator keys using the provided password
3) Split the validator's secret key into multiple shares based on the number of operators
4) Encrypt each keyshare with the respective operator's RSA public key
5) Construct a payload with all necessary data for the SSV network
6) Write the output data to a file in the specified location

The tool uses threshold cryptography to ensure that validation can continue even if some operators go offline, while maintaining security against potential attackers.

# Example Usage Format
```bash
# Manual key splitting
anchor keysplit manual \
  --keystore-path /path/to/validator_keystore.json \
  --password your_keystore_password \
  --owner 0x123... \
  --operators 1,2,3,4 \
  --output-path /path/to/output.json \
  --nonce 0 \
  --public-keys key1,key2,key3,key4

# Onchain key splitting
anchor keysplit onchain \
  --keystore-path /path/to/validator_keystore.json \
  --password your_keystore_password \
  --owner 0x123... \
  --operators 1,2,3,4 \
  --output-path /path/to/output.json \
  --rpc https://eth-hoodi.provider.com \
  --network Hoodi
```

# Command Options
## Shared
- `--keystore-path PATH`: Path to the validator keystore file
- `--password PASSWORD`: Password for the validator keystore
- `--owner ADDRESS`: EOA address that owns the validator
- `--output-path PATH`: Path for output file
- `--operators IDS`: Comma-separated list of operator IDs (must be 4, 7, 10, or 13 operators)

## Manual Mode Options
- `--nonce NONCE: Nonce`: for the owner address
- `--public-keys KEYS`: Comma-separated list of RSA public keys for the operators
## Onchain Mode Options
- `--rpc ENDPOINT`: RPC endpoint to access L1 data
- `--network NETWORK`: Ethereum network (Holesky or Hoodi - Mainnet is not supported)

# Output
The tool generates a JSON file compatible with the SSV validator registration webapp.
